Roadmap

Travel downward through the production plan.

The first public release is planned as a stable community build, not every future feature at once. The path below tracks a realistic six-month route from scope to public release candidate.

  1. Month 01 / Pre-production Theater foundation and naval scope

    Lock the playable vision: island chains, sea-lane logic, ship class roles, aviation needs, target categories, landing areas, mission cadence, and server limits. This phase turns ambition into a clear build list.

  2. Month 02 / Asset pipeline Boat and warship production pass

    Prepare hull models, collision geometry, LODs, materials, damage zones, wake references, turrets, launcher sockets, crew points, and prefab structure. Each vessel moves through import, validation, and first playable placement.

  3. Month 03 / World integration Ocean, ports, and operation spaces

    Shape water as terrain: routes, dock areas, amphibious corridors, objective islands, recon lanes, carrier or fleet staging, coastal approach logic, lighting, weather, and atmospheric readability for long-range engagements.

  4. Month 04 / Systems testing Movement, weapons, sensors, and flight control

    Test ship handling, turning radius, acceleration, fire-control behavior, missile profiles, helicopter support, jet operations, HOCAS-compatible control profiles, replication, dedicated-server behavior, and performance under real scenario load.

  5. Month 05 / Closed community trials Scheduled operations and balance correction

    Run trusted community playtests focused on navigation clarity, mission pacing, server stability, survivability, weapon ranges, logistics pressure, spawn logic, objective value, and whether the naval layer creates repeatable battles.

  6. Month 06 / Release candidate Workshop packaging and public deployment

    Finalize metadata, dependency notes, server instructions, screenshots, launch scenario, known issues, changelog, and showcase media. The first release should be focused, stable, and ready to expand through updates.

Player Q/A

What this project is trying to become.

No. The goal is a naval theater where ships, aircraft, amphibious movement, recon, and coastal objectives all affect the match. Vessels should create decisions, not sit offshore as scenery.

The design pulls from naval command pacing, modern strike tension, and serious flight-control expectations while staying practical for multiplayer operations. It is inspiration, not a port or copy of another title.

The plan includes jets, helicopters, recon, strike roles, and HOCAS-compatible control profiles where the engine and input stack allow it. The goal is credible control depth without making every pilot learn a full study-sim cockpit.

Large vessels need more than a model. They require collision, LODs, damage setup, launch points, weapon logic, crew positions, water interaction, wakes, networking, server performance checks, and mission logic that gives the vessel a reason to exist.

The working target is a six-month route toward a stable community release candidate. Closed testing should happen before public packaging so the first release feels intentional, readable, and worth returning to.
