Roadmap

Six months from scope lock to release candidate.

The first public build should be focused and stable. This track moves from hard scope, to imported assets, to playable water, to closed operations, then Workshop packaging.

  1. Month 01 / Pre-production Scope lock and theater foundation

    Define island chains, sea lanes, ship roles, aircraft needs, landing areas, target categories, mission cadence, and server limits. The result is a build list the team can actually ship.

  2. Month 02 / Asset pipeline Hull, weapon, and prefab pipeline

    Prepare hulls, collision, LODs, materials, damage zones, wake references, turrets, launch sockets, crew points, and prefab structure. Each vessel moves from import to first playable placement.

  3. Month 03 / World integration Ocean routes, ports, and objectives

    Build the playable water: dock areas, amphibious corridors, objective islands, recon lanes, fleet staging, coastal approaches, lighting, weather, and readable long-range contact points.

  4. Month 04 / Systems testing Movement, weapons, sensors, and controls

    Test ship handling, missile behavior, helicopter support, jet operations, HOCAS-compatible profiles, replication, dedicated-server behavior, and performance under scenario load.

  5. Month 05 / Closed community trials Closed operations and balance passes

    Run trusted playtests for navigation clarity, mission pacing, stability, survivability, weapon ranges, logistics pressure, spawn logic, objective value, and replayable naval fights.

  6. Month 06 / Release candidate Workshop package and launch brief

    Finalize metadata, dependencies, server notes, screenshots, launch scenario, known issues, changelog, and showcase media. The first release lands focused, stable, and ready to expand.
