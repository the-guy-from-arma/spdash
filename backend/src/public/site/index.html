Mid Pacific Naval Theater

Thunder Buddies Studios is building an ambitious maritime theater for Arma Reforger, designed to make open water, warship command, reconnaissance, amphibious support, and large-scale naval conflict matter.

The missing battlefield

For years, Arma has delivered infantry combat, armored warfare, and aviation.

True naval warfare has never had a permanent place on the battlefield. The Mid Pacific Naval Theater is designed to change that by turning the ocean from empty space into a living operational front.

Six-month development path

Testing to community release

A reasonable first public deployment is not every naval feature at once. The target is a stable community build over six months: production assets, playable world space, internal systems testing, closed community trials, then a Workshop-ready release candidate.

Scope Assets World Systems Closed Tests Release
  1. Month 01 / Pre-production Theater foundation and naval scope

    Lock the Mid Pacific Naval Theater design: island chains, engagement distances, sea-lane purpose, mission flow, target roles, faction needs, and what each ship class must contribute. This is where the team decides what is feasible inside Reforger and what must be staged for later updates.

  2. Month 02 / Asset pipeline Boat and warship production pass

    Build or clean hull models, collision geometry, LODs, materials, textures, sockets, turrets, launch points, radar details, wake references, and damage regions. Assets move through Enfusion import, prefab setup, config work, and first in-game placement before any serious gameplay testing begins.

  3. Month 03 / World integration Ocean, ports, and operation spaces

    Shape the playable maritime world: spawn zones, docks, amphibious landing areas, reconnaissance corridors, objective islands, server-safe staging areas, and visual atmosphere. This month is about making the water feel like terrain, not a dead border around the infantry fight.

  4. Month 04 / Systems testing Movement, weapons, sensors, and multiplayer behavior

    Internal tests focus on ship handling, turning radius, acceleration, AI/crew assumptions, missile or fire-control behavior, helicopter interaction, amphibious support, network replication, and dedicated-server stability. The goal is a playable vertical slice, not final balance.

  5. Month 05 / Closed community trials Discord playtests and balance correction

    Trusted community testers run scheduled operations. Feedback goes into navigation clarity, mission pacing, server performance, ship survivability, weapon ranges, spawn logic, logistics pressure, and whether naval objectives actually create fights players want to repeat.

  6. Month 06 / Release candidate Workshop packaging and public deployment

    Prepare Workshop metadata, dependency notes, server instructions, known-issues list, screenshots, showcase video, changelog, and launch-night scenario. A reasonable first public release is a stable community build with focused naval gameplay, followed by expansion updates rather than every planned feature at once.
